# 機能要件一覧

| id | functionName | description | relatedRequirement | priority |
| --- | --- | --- | --- | --- |
| FR-001 | ユーザー会員登録・認証機能 | 新規ユーザーのアカウント登録、ログイン、ログアウト、パスワードリセットを行う画面および制御機能。※要確認：大学共通のシングルサインオン（SSO）や学認（GakuNin）などの外部認証連携要否（技術要件・技術制約、議事録・会議メモ）。 | BR-USR | 高 |
| FR-002 | プロフィール・APIキー（BYOK）設定機能 | ユーザーがプロフィール情報を更新し、自身の外部AI APIキー（OpenAIやAnthropic等）を入力・登録・検証する画面。※要確認：BYOKモデルに基づくAPIキーの暗号化保存基準および個人情報保護ポリシー（技術要件・技術制約、業務要件・ビジネスルール）。 | BR-USR | 高 |
| FR-003 | スタートアッププロジェクト管理機能（CRUD） | 起業のアイデア単位でプロジェクトを新規作成、一覧表示、詳細編集、削除する画面機能。※要確認：1ユーザーあたりが作成可能なプロジェクト数の上限値およびアーカイブ・削除時の論理削除/物理削除の定義（業務要件・ビジネスルール）。 | BR-PRJ | 高 |
| FR-004 | AIスタートアップアイデア検証機能 | 入力されたビジネスアイデアに基づき、外部AIを利用して実現可能性、新規性、ターゲット市場の親和性を多角的に検証・フィードバックする画面。※要確認：分析時に適用するプロンプトエンジニアリングの構造および評価項目セット（ドメイン知識）。 | BR-IV | 高 |
| FR-005 | リーンキャンバス自動生成機能 | 登録されたアイデア情報を基に、AIが自動でリーンキャンバスの9要素（課題、解決策、主要指標等）を下書き生成する機能。※要確認：起業手法フレームワークとしての9要素に適合する入力パラメータ定義（ドメイン知識）。 | BR-LC | 高 |
| FR-006 | リーンキャンバス編集・保存機能（CRUD） | AIが生成したリーンキャンバスの内容をユーザーが手動で編集、追加、削除し、保存する画面。バージョン管理（履歴保存）にも対応する。※要確認：世代履歴の最大保存数および比較画面の有無（業務要件・ビジネスルール）。 | BR-LC | 高 |
| FR-007 | SWOT分析・戦略提案生成機能 | 強み(S)、弱み(W)、機会(O)、脅威(T)をAIがマトリクス化し、さらにそれらを掛け合わせた「クロスSWOT分析」から戦略提案を自動生成する画面機能。※要確認：提案ロジックのフレームワーク整合性（ドメイン知識）。 | BR-SW | 高 |
| FR-008 | SWOT分析結果編集・履歴管理機能（CRUD） | 生成されたSWOT要素および提案戦略をユーザーが手動で編集・カスタマイズし、データベースに上書きまたは新規保存する機能。※要確認：バージョンごとの比較表示機能の要否（議事録・会議メモ）。 | BR-SW | 高 |
| FR-009 | MVPロードマップ自動生成機能 | AIを活用し、アイデア実現に必要な主要機能を洗い出し、MoSCoW法（必須・推奨等）により優先順位を設定して初期ロードマップを自動生成する画面機能。※要確認：開発期間・規模を見積もるためのアルゴリズムおよび設定項目（ドメイン知識）。 | BR-MR | 高 |
| FR-010 | MVPマイルストーン・タスク管理機能（CRUD） | 生成されたロードマップ内のマイルストーンおよび個別タスクの作成、並び替え、期日編集、ステータス（未着手・進行中・完了）更新、削除を行う画面。※要確認：かんばんボード形式による視覚的操作要件（議事録・会議メモ）。 | BR-MR | 中 |
| FR-011 | PDFレポート出力機能 | プロジェクトに蓄積されたリーンキャンバス、SWOT分析、MVPロードマップの全データを集約し、PDF形式で一括エクスポートする機能。※要確認：日本語フォント埋め込み、レイアウト設計、およびファイルサイズ制限（技術要件・技術制約）。 | BR-EXP | 高 |
| FR-012 | 管理者：ユーザー・権限管理機能（CRUD） | システム管理者がユーザー一覧を表示し、アカウント状態の監視、権限（学生/管理者等）変更、不要アカウントの削除を行う機能。※要確認：メンター等、追加が予定されている複数ロールのパーミッション設定方針（業務要件・ビジネスルール）。 | BR-ADM | 中 |
| FR-013 | 管理者：AIプロンプト・モデル制御機能（CRUD） | 管理画面から各分析プロセスで利用するAIモデル（GPT-4、Claude3等）の指定や、プロンプトの更新・管理を可能とする機能。※要確認：プロンプト更新時のテスト実行機能の要否（技術要件・技術制約）。 | BR-ADM | 中 |
| FR-014 | 管理者：利用状況統計ダッシュボード機能 | 累計ユーザー数、プロジェクト稼働数、PDF生成頻度、外部API接続ログをグラフ形式で可視化する管理画面機能。※要確認：大学・講義ごとの集計区分定義（議事録・会議メモ）。 | BR-ADM | 低 |
| FR-015 | 一時生成ファイル自動削除バッチ | PDFエクスポート時にサーバーまたはクラウドストレージへ一時作成されたファイルを、生成後一定時間経過（例: 24時間）した時点で自動的にクリーンアップする夜間バッチ処理。※要確認：ストレージ容量制限および削除ポリシー（技術要件・技術制約）。 | BR-SYS | 中 |
| FR-016 | AIモデレーション・不適切コンテンツ検知バッチ | ユーザーが登録したプロジェクトの入力内容やAIからの出力内容について、定期的なテキストフィルタリングや規約違反フレーズの監視を行い、不適切なコンテンツを管理者に警告通知するバッチ処理。※要確認：大学推奨のモデレーション基準（業務要件・ビジネスルール）。 | BR-SEC | 低 |
| FR-017 | プロジェクト共同編集機能（拡張機能） | チーム開発用の複数人同時閲覧・編集制御、および権限管理機能。※要確認：WebSocketなどのリアルタイム通信技術の適用および同時競合発生時のロック制御仕様（技術要件・技術制約、議事録・会議メモ）。 | BR-COL | 低 |
| FR-018 | メンターフィードバック・評価登録機能（拡張機能） | 招待された外部メンターが、学生のプロジェクト（リーンキャンバスやロードマップ）を閲覧し、指摘コメントや評価を登録・通知する機能（CRUD）。※要確認：評価基準となるルーブリックや通知機能の連携仕様（ドメイン知識、業務要件・ビジネスルール）。 | BR-MNT | 低 |
